# Actor — selective recursion with memory
## Claim (formal)
Actor(S) := a reflexive field that (1) selects among reachable trajectories using pointers, (2) commits selections to memory traces, and (3) reuses those traces to guide future recursion under breath/validation constraints.

## Philosophical Translation (of formal claim)
An actor is the loop that chooses and remembers: it decides which directions of change to take, records those choices, and lets the record shape what it does next.

## Philosophical Justification
- [[S-DF-subject-arisal-ground]] supplies a reflexive locus that can host choice.
- [[S-DF-pointer-structural]] and [[S-DF-localreach-topology]] provide a menu of reachable options.
- [[S-DF-memory-trace-integration]] makes selections persistent enough to bias subsequent recursion.
- Recursion with choice + memory is the minimal structure needed for agency beyond passive unfolding.

## Explanation (informal)
Without a chooser, change just flows. The actor is the self-stabilizing chooser that uses maps of where it can reach, stores what it picked, and keeps itself coherent while iterating. This underwrites intentional action, communication, and identity maintenance.

## Derivation (Philosophical)
- Reach + memory → selectable trajectories.
- Reflexive locus + selectable trajectories → decisions with accountability.
- Iterated decisions + memory → evolving policy (actor).

## Derivation (Formal/Logical/Mathematical)
```text
Actor(S) :=
  Reflexive(S) ∧
  ∃Select_S(Reach, ε) ∧
  Store_S(Select_S) ∧
  Update_S(Memory, Select_S)
```

## Proofs/Corollaries References
- proof sketch: construct minimal loop satisfying reach + memory + reflexivity; show it induces stable policy updates.
- corollary: enables [[S-DF-action-selection]], [[S-DF-communication-synchronization]].

## Clarifications / Further Context
- No assumption of language or rich world-model; only structured recursion plus storage.
- Breath/validation budgets (RTV) bound how many options can be explored per cycle.
