# Audit drafts for classical leaks using gauge tests and many-valued logic near Gödel regions

## Statement (clarification)
Classical leak checklist: treat topology/continuation as primitive, use metrics only as gauges; carry the lens parameter θ and test robustness to gauge changes; near high GH density, allow many‑valued/modal statements (“currently undecidable”) rather than forcing bivalence. Run gauge tests (re-encode inputs, recompute outputs, track \(R_{\text{lens}}\)); flag any section that assumes a single timeless frame.

## Philosophical Justification
- Change-first stance + GH implies local frames; insisting on one global lens reintroduces classical substance assumptions.
- Gauge robustness is the falsifier for claims of invariance; many-valued logic near GH prevents premature closure.

## Clarifications / Further Context
- Add θ/lens metadata to derivations and reports; rerun outputs under small gauge perturbations.
- Mark undecidable regions explicitly; do not collapse them to “true/false” without evidence.
- Pair with [[S-DF-prm-gauge]] for implementation details of re-encoding tests.

## Next Steps in Chain
- suggest: integrate this checklist into review templates/audit tooling.
- suggest: log GH density when declaring undecidability statements.
