# Define holes relative to (S,L,T) and measure creative vs mirage density

AI_13 sharpens the definition: for a system \(S\), language \(L\), and solver \(T\), a property \(P\) is a hole when \(P\) has determinate truth in \(S\) but \(T\) cannot decide it, and a bounded extension \((L',T')\) makes \(P\) accessible. Hole density can be quantified by compression gain per unit language expansion (\(\rho\)) or by pre-registered MDL drops. Distinguish creative holes (one extension that slashes residuals across contexts) from mirage holes (overfitting a single dataset). Practical detectors include compression gaps, brittleness/transfer failure, and innovation pressure (fitness benefit of new representational machinery). The guidance replaces vague “holes in reality” talk with auditable, MDL-anchored metrics.
