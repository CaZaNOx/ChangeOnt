# Arisal — change-led becoming-present
## Claim (formal)
Arisal(x) := Δ brings x into presence; becoming-present is a structural capability of change. Subject-conditioned arisal (experienced arisal) is a special case; arisal does not presuppose a subject.

## Philosophical Translation (of formal claim)
Becoming is not added to change; it is what change does — bringing something to presence.

## Philosophical Justification
- From [[S-FT-immediate-datum]], Δ is unavoidable; “newness” is therefore primitive.
- Calling something “present” presupposes that it was not already so; the gap between “not yet” and “now” is arisal.
- No extra machinery (memory, time parameter) is required: Δ itself supports the distinction.

## Explanation (informal)
Arisal names the basic move where change instantiates a new configuration. Later structures (identity, subject, priors) refine how arisal is tracked, but the primitive capacity is already implied by Δ.

## Derivation (Philosophical)
- If Δ ≠ ∅, then some predicate flips from false to true; that flip is arisal.
- Subjective arisal adds a bearer of experience but inherits the same structural flip.

## Derivation (Formal/Logical/Mathematical)
```text
Δ ⇒ ∃x : ¬Present(x) ∧ Δ(x) ⟹ Present(x)
```

## Proofs/Corollaries References
- corollary: arisal enables defining generalized occurrences ([[S-DF-point-occurrence]]).

## Clarifications / Further Context
- Distinguish general arisal from subject-experienced arisal; both are grounded in Δ.
- Arisal sets up later notions of priors, paths, and stabilization energy by providing the “arrival” that must be integrated.
