# Keep a formula derivation ledger mapping each tool to CO layers and assumptions

AI_13’s audit from line 29834 onward shows how to police every imported formula: for each object (path algebra, coarse-grain, probability, KL, Fisher metric, Lyapunov, MDL, etc.) list the CO layer ladder it requires, the extra assumptions (support, smoothness, ergodicity), a DS justification (“what the doubting subject experiences”), its status (native vs. imported), and a falsifier (where the assumption fails). Coupled with the “minimal syntax” sheet—paths, coarse-grain operator \( \mathcal{C}_\epsilon \), counting, KL/Fisher metrics, Lyapunov stability, attention reweighting, and hedge/probe/jump gap policy—you get a cheat-sheet linking every equation back to declared change-first primitives. This ledger prevents smuggled classical baggage: whenever a new formula appears, you must place it on the ladder, state the assumptions, and note the test that would break it.
