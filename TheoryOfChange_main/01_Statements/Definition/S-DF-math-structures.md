# Math structures for CO — selection principles
## Claim (formal)
Select structures that: (i) respect LocalReach topology, (ii) admit Tx representation, (iii) support invariants/similarity, and (iv) integrate with RTV/audit protocols.

## Philosophical Translation (of formal claim)
We pick math that fits the change — not the other way around.

## Philosophical Justification
- [[S-DF-localreach-topology]] grounds neighborhood/continuity; chosen structures must honor that substrate.
- [[S-DF-tx-operator]] requires structures stable under frame shifts.
- [[S-DF-identity-invariants]] and [[S-DF-rtv-operator]] demand similarity/invariant handling and recursive validation compatibility.
- Selecting structures by these criteria avoids smuggling in classical artifacts and keeps tools accountable to Δ.

## Explanation (informal)
Instead of assuming Euclidean or Boolean defaults, we select algebra/topology/logic that respect reach, identity, and validation dynamics. This prunes structures that break under Tx or ignore invariants.

## Derivation (Philosophical)
- Define admissible class C of structures satisfying the four criteria.
- Prefer minimal C that still supports required operators (Sim, Eval, Gauge).

## Proofs/Corollaries References
- corollary: informs [[S-DF-gauge-alignment-field]] (alignment fields), [[S-DF-metric-space-emergent]] (derived metrics).

## Clarifications / Further Context
- Candidate structures: quantales for Eval, pseudometrics from Sim, sheaves over LocalReach for gluing invariants.
- Auditability: structures must expose failure modes under RTV (collapse, divergence).
