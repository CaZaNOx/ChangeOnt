# Failure memory treats past chats as negative data

The failure memory block must store prior AI chats as _negative data_ that informs robustness tests: every recorded failure becomes a template for what to avoid rather than a source of stylistic imitation. The failure memory should be quarantined (no direct reuse) and actively contrasted with current traces so that the agent learns what not to repeat when building new `DF/CR` anchors. This acts as a drift guard and prevents replaying collapsed or low-Î” sequences.
