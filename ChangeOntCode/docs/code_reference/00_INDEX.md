# Code Reference Index

This section mirrors the code tree and explains each major folder/subsystem.

Its purpose is different from the conceptual/kernel docs.

- Conceptual docs define the target-state architecture and contracts.
- Code reference docs explain what each code subsystem does, why it exists, and where it fits in the runtime.

## Read this section when you need to know:
- what a folder/file does,
- what calls it,
- what it calls,
- what contract it satisfies,
- whether it is core / optional / experimental / legacy.
