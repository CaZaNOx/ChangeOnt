# Path logic needs homotopy classes, bend operators, and return maps beyond Markov chains

AI_13 explains why change ontology insists on path logic rather than first-order Markov models. Paths \(\gamma:[0,1]\to\mathcal{X}\) carry knot-type and homotopy information (walls/holes) that Markov states cannot encode without exploding state space. Operators act on whole paths: ChangeBudget \(L[\gamma]=\int \|\dot \gamma(s)\|\,ds\), practice/relax maps \(T\) contract small bends back to an attractor, feasibility projectors \(\Pi_{\mathcal{C}}\) enforce walls, and homotopy classes capture hysteresis. Absorbing small bends means \(T\) is a contraction within each path class (Banach fixed point), so repeated practice returns you to the loop. Markov chains can only emulate this by embedding massive history; path logic treats it natively, preserving the link between innovation zones (walls/holes where bends blow up) and operator design.
