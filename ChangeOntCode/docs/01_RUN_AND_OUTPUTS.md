# Run and Outputs

## Run the suite (all families)
Exact command (default config):
- `python -m experiments.suite_cli`

Run with a specific suite config:
- `python -m experiments.suite_cli --config experiments/configs/suite_demo.yaml`

Suite config files (used by `suite_cli`):
- `ChangeOntCode/experiments/configs/suite_all.yaml` (families, modes, seeds, agents, envs)
- `ChangeOntCode/experiments/configs/co_agents/co_agents_selection.yaml` (CO variants injected across families)

Config toggles:
- `inject_co: true|false` (default true; disables CO injection when false)
- `co_manifest: <path>` (optional override; resolved relative to the suite config file)

What the suite does:
- loads the suite config (default `experiments/configs/suite_all.yaml` unless `--config` is provided)
- generates per-run configs under `ChangeOntCode/outputs/suite/tmp/<family>/<mode>/...`
- calls each family runner as a subprocess
- writes per-mode and per-family summaries plus a suite summary

Note on output root:
- When `--config` is used, `out_root` is automatically suffixed with a timestamp: `_YYYY-MM-DD-HH-MM-SS`.

## Run a single family (exact commands)
Bandit:
- `python -m experiments.runners.bandit_runner --config <yaml>`

Maze:
- `python -m experiments.runners.maze_runner --config <yaml>`

Renewal:
- `python -m experiments.runners.renewal_runner --config <yaml>`

Notes:
- Per-runner configs are generated by the suite under `ChangeOntCode/outputs/suite/tmp/`.
- If you want to run without a config file, each runner supports CLI fallbacks.

## Smoke runs (seed=0)
Use the CLI fallbacks (seed=0) and reference existing suite-generated config examples for shapes/values.

Bandit (values mirror `suite_all.yaml` “easy” mode):
- Command: `python -m experiments.runners.bandit_runner --probs 0.10,0.20,0.80 --horizon 1000 --agent ucb1 --seed 0 --out ChangeOntCode/outputs/verify/bandit_s0`
- Config example shape: `ChangeOntCode/outputs/suite/tmp/bandit/easy/ucb1_s7.yaml`

Maze (values mirror `suite_all.yaml` “maze_9x9” mode):
- Command: `python -m experiments.runners.maze_runner --maze environments/maze1/spec_9x9.yaml --episodes 1 --agent bfs --out ChangeOntCode/outputs/verify/maze_s0`
- Config example shape: `ChangeOntCode/outputs/suite/tmp/maze/maze_9x9/bfs_s7.yaml`

Renewal (values mirror `suite_all.yaml` “clean” mode):
- Command: `python -m experiments.runners.renewal_runner --config ChangeOntCode/outputs/suite/tmp/renewal/clean/last_s7.yaml`
- To run seed=0, copy the config above and change `seed: 0` (no repo config uses seed=0 by default).

## Outputs layout
Suite outputs (default root is `ChangeOntCode/outputs/suite/`):
- Per-run: `ChangeOntCode/outputs/suite/<family>/<mode>/<agent>_s<seed>/`
- Per-mode summary: `ChangeOntCode/outputs/suite/<family>/<mode>/_summary/summary.csv` (+ `summary.png`)
- Per-family summary: `ChangeOntCode/outputs/suite/<family>/_summary/combined_summary.csv` (+ `combined_summary.png`)
- Suite summary: `ChangeOntCode/outputs/suite/summary/overall_summary.csv` and `overall_stoa_vs_co.csv` (+ `overall_stoa_vs_co.png`)

Single-run outputs (from each runner):
- `metrics.jsonl` — JSONL event log
- `budget.csv` — budget counters (flops/memory/etc.)
- `quick_plot.png` — best-effort plot (if plotting deps available)

## metrics.jsonl schema (high level)
Each line is a JSON object. Typical record patterns include:
- Header record: `record_type`, `runner`, `seed`, `env`, `agent`, plus `episodes` or `mode` where applicable.
- Metric record: `metric`, `t` or `episode`, and metric-specific fields like `value`, `arm`, `reward`, `cum_reward`.
- CO telemetry: `metric="co_debug"` with fields such as `action`, `co_policy`, `co_weight`, `co_bus_votes`,
  plus counters (`birth_events`, `merge_events`, `split_events`, `bend_triggers`, `prototype_count`, `class_count`, `cap_hits`)
  and signals (`last_d`, `identity_ok`, `ghvc_pressure`, `ghvc_mdl_gain`).
