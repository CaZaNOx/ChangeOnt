# Attractors require recurrent compressibility, not literal loops

AI_13 clarifies that subject-level attractors need recurrently compressible patterns: features returning within a tolerance \(T_{\text{ret}}\) shorter than forgetting but slower than drift, with transverse contraction (negative Lyapunov exponents). Memory defines equivalence classes \(x \sim y\) when \(\|\phi(x) - \phi(y)\| < \varepsilon\) after \(K\) confirmed returns. Drift faster than consolidation destabilizes the code. The “collapse to a 2D circle” test only catches limit cycles; many attractors are symbolic/code manifolds, spirals/ratchets, or loops in quotient spaces. Calling them all loops hides the distinction between true closure and motif recurrence with drift.
