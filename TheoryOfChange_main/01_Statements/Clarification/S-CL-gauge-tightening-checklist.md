# Freeze the gauge, preregister endpoints, and audit models

AI_13’s actionable checklist (lines 17364‑17379) says to: (1) freeze \(G\) by specifying the exact mathematical class of allowed \(\phi\) (isometries for \(d\), Lipschitz-bounded monotone warps for affect); (2) preregister primary endpoints (commuting counterfactuals quantified via KL between predicted vs observed transitions) and secondary metrics (STE, \(\nabla A\) alignment, holonomy gaps); (3) plan power across time scales so kernels \(K_u\) are fitted with uncertainty estimates; (4) build a lesion library for \(J\) components (dual-task, sleep restriction, pharmacological manipulations) with predicted signatures; (5) publish model-audit code reporting gauge residue and each invariant’s contribution. Without these guardrails the gauge program risks overfitting or unfalsifiable flexibility.
