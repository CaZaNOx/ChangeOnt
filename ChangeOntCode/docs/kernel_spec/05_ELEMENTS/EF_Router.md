# EF_Router

## Purpose

EF_Router is a **reserved future element role** for explicit routing between internal mechanism regimes.

At present it is not part of the required v1 canonical semantic kernel.

---

## Element Role

If used later, EF_Router would be an **element** that groups together routing-related mechanism logic.

It is not:
- a primitive
- a translator
- the ActionHead
- the full header/meta-header layer

---

## Current Binding Status

For v1:
- EF_Router is not required
- EF_Router should not be silently introduced as a hidden dependency in the canonical path
- EF_Router remains future/non-central unless explicitly reactivated by updated doctrine/spec

---

## Why it exists conceptually

The project may later need an explicit routing mechanism between:
- different internal semantic regimes
- different blends of CO/classical behavior
- different stability-controlled pathways

That role is conceptually valid, but not yet binding for the current kernel slice.

---

## Forbidden

Until explicitly reintroduced:
- do not rely on EF_Router as a required canonical component
- do not hide critical kernel behavior inside it

---

## Current Status

Reserved future role only.
