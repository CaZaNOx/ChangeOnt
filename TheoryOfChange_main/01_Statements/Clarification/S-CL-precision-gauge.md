# Precision gauge saves compute vs fixed-digit classical Pi

CO treats precision as a gauge: each holographic tolerance τ selects the minimal digits `n` needed to keep `|π−π_n| ≤ τ`. Headers only raise `n` when error or dynamic structure demands it (collapse header H/loop_score). Classical fixed-digit approximations must provision for the worst case; CO’s precision gauge reduces cumulative digits-seconds while maintaining accuracy, especially on tasks where tolerance oscillates. This is the precise Pi example from AI_15: when τ drops the gauge automatically raises precision; when τ relaxes the gauge coarsens without manual tuning. Premature adoption of fixed digits would collapse the approach to the classical heuristic belt.
