# Falsify the identity bridge by targeting the thick-present tuple

The bridge law “experience(t) = (G^*, U, B, A, \tau)” only holds if every component co-moves with the reported phenomenal character. AI_13 lines 16586‑16640 lay out practical kill criteria: (K1) produce reliable shifts in felt vividness, mineness, or valence without corresponding changes in measured \(G^*, U, B, A, \tau\); (K2) engineer two states with matched tuples yet diverging downstream learning or behavioral updates, showing the tuple is not the driver; (K3) create systems that retain metacognition, affect, and reportability while suppressing the self-gradient \(G^*\). Experimental protocols—varying \(\tau\) via masking, clamping \(U\) via neuromodulators, ablating \(A\) with TMS/sedation—supply the empirical hooks. If any kill criterion is satisfied, the identity thesis fails; otherwise the anti-zombie constraint survives repeated attempts at falsification. The same segment insists on pre-registered intervention suites and simulator agents: dial τ/B/A/U independently, reproduce attentional-blink/rivalry effects in AI, and publish “claim + tests + kill criteria” sheets so skeptics can rerun the protocol.
